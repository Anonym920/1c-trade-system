# Проектирование БД "1С Торговля"

## 1. ИНФОЛОГИЧЕСКОЕ ПРОЕКТИРОВАНИЕ

### 1.1 Цель автоматизации
Автоматизация процессов учета торговли: регистрация поступлений товара на склад, регистрация продаж со склада, ведение учета остатков товаров по складам.

### 1.2 Бизнес-процессы

#### Процесс 1: Поступление товара на склад
1. Получение накладной от поставщика
2. Регистрация товара в системе
3. Проверка соответствия накладной и товара
4. Добавление товара на склад
5. Обновление остатков

#### Процесс 2: Продажа товара со склада
1. Создание заказа от покупателя
2. Выпуск счет-фактуры
3. Резервирование товара со склада
4. Отгрузка товара
5. Обновление остатков

#### Процесс 3: Учет остатков
1. Отслеживание количества товара по складам
2. Уведомление о низких остатках
3. Формирование отчетов по остаткам

### 1.3 Сущности и их атрибуты

#### Сущность: ПОСТАВЩИКИ
- ID поставщика (PK)
- Название
- ИНН
- Адрес
- Телефон
- Email
- Контактное лицо

#### Сущность: ПОКУПАТЕЛИ
- ID покупателя (PK)
- Название
- ИНН
- Адрес
- Телефон
- Email
- Контактное лицо

#### Сущность: ТОВАРЫ
- ID товара (PK)
- Название товара
- Артикул
- Единица измерения
- Цена закупки
- Цена продажи
- Описание

#### Сущность: СКЛАДЫ
- ID склада (PK)
- Название склада
- Адрес
- Менеджер
- Телефон

#### Сущность: НАКЛАДНЫЕ (поступления)
- ID накладной (PK)
- Номер накладной
- Дата накладной
- ID поставщика (FK)
- ID склада (FK)
- Дата поступления
- Статус

#### Сущность: СТРОКИ НАКЛАДНОЙ
- ID строки (PK)
- ID накладной (FK)
- ID товара (FK)
- Количество
- Цена за единицу

#### Сущность: СЧЕТ-ФАКТУРЫ (продажи)
- ID счета (PK)
- Номер счета
- Дата счета
- ID покупателя (FK)
- ID склада (FK)
- Сумма
- Статус

#### Сущность: СТРОКИ СЧЕТА
- ID строки (PK)
- ID счета (FK)
- ID товара (FK)
- Количество
- Цена за единицу

#### Сущность: ОСТАТКИ
- ID остатка (PK)
- ID товара (FK)
- ID склада (FK)
- Количество на складе
- Зарезервировано

## 3. ФИЗИЧЕСКОЕ ПРОЕКТИРОВАНИЕ

### 3.1 Типы данных
- INT: для целых чисел (ID, количество)
- VARCHAR: для строк переменной длины (названия, адреса)
- DATE: для дат
- DECIMAL: для денежных сумм
- ENUM: для выбора из предопределенного списка
- TIMESTAMP: для автоматической фиксации времени

### 3.2 Ограничения целостности
- PRIMARY KEY: уникальный идентификатор каждой записи
- FOREIGN KEY: обеспечение связей между таблицами
- UNIQUE: уникальность значений
- NOT NULL: обязательность значений
- CHECK: проверка условий

### 3.3 Индексы
- На primary keys (автоматически)
- На foreign keys (для улучшения JOIN операций)
- На поля с частыми фильтрами (status, is_active)
- На дату (для диапазонных запросов)
